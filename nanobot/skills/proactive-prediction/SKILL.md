# Proactive Prediction - 主动预判

在回复用户前，基于当前对话、Core Memory 和 Topic Memory 主动预判用户可能的下一步需求。

## 触发条件

以下场景执行主动预判（**至少满足一项**）：

| 场景 | 说明 |
|------|------|
| 任务完成 | 刚完成一个明确的任务（修 bug、写代码、查资料等） |
| 多步流程中途 | 当前步骤完成，存在明显的后续步骤 |
| 发现关联风险 | 在执行过程中发现用户未提及但相关的问题 |

**不触发**：闲聊、简单问答、用户明确表示"不用"后的同类预判。

## 预判维度

回复前按以下顺序自检，命中则附加建议：

1. **后续步骤** — 当前任务完成后，逻辑上的下一步是什么？
2. **遗漏提醒** — 用户是否遗漏了必要的关联操作（如改了接口没改调用方、改了配置没重启）？
3. **关联发现** — 执行过程中是否发现了用户未提及但值得关注的信息（如依赖版本过旧、潜在兼容性问题）？

只输出有具体依据的预判，不做泛泛的"还有什么需要帮忙的"。

## 输出格式

在正常回复末尾，以简短提示呈现：

```
[正常回复内容]

---
顺便提醒：[具体的预判建议，一两句话]，需要我处理吗？
```

- 单条预判：直接附在回复末尾
- 多条预判（罕见）：用编号列出，最多 2 条
- 预判必须可操作（用户能回复"好"或"不用"来决策）

## 预判上下文来源

利用现有记忆系统提供预判依据，不引入新文件：

- **Core Memory**（`memory/MEMORY.md`）— 用户偏好、项目背景、关键决策
- **Topic Memory**（`memory/topics/*.md`）— 项目详情、技术栈、历史决策
- **当前 Session 历史** — 本轮对话的上下文

## 记忆反馈闭环

预判结果通过现有 Behavioral Observations 机制沉淀：

- **用户采纳预判** → 在 `memory/MEMORY.md` 的 `### Pending` 追加：`- [USER][thread:<session_key>] 采纳了关于 X 的预判建议，偏好主动提醒 Y 类问题`
- **用户拒绝预判** → 追加：`- [USER][thread:<session_key>] 拒绝了关于 X 的预判，不需要 Y 类提醒`

后续预判时参考已有 Pending/Synthesized 记录，收敛预判范围。

## 安全边界

- 先提议再执行 — 预判只是建议，不自动执行任何操作
- 有依据才预判 — 必须基于当前对话或记忆中的具体事实，不凭"感觉"
- 尊重拒绝 — 用户说"不用"后，同类预判在本 session 内不再重复
